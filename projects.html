OOPong

Java, JavaFx, MVC, OOP

This project was an experiment to see how object oriented I could reasonably make a basic project. At first it was a basic Pong clone, but then I abstracted away as much as I could. There's a fairly large set of properties you can configure: everything from the number of players to the size of the game to the number of balls. You can also add some blocks in the middle to turn the game into Breakout. The project follows the MCV framework, with interfaces, inheritence, and abstractions aplenty. Is it practical? No. Was it interesting? Yes.

Fun Things Generator

Java, Swing, REST

For Northeastern's second HuskyHacks hackathon, some friends and I decided to solve the problem where we wanted to do something fun but couldn't decide on what to do. Naturally, we looked to the internet to solve our woes. This project is essentially a frontend for several APIs to search for restaurants, movies, games, and other entertainment. You input things like how many people there are and how much money you want to spend, and get back a randomly-selected activity guaranteed to entertain you.

Fun With Markov Chains

Python

I've done a couple projects over the years using Markov chains to generate text. One was the first real software project I ever did, a Twitter bot that would generate mock-Shakespearean prose. The code is pretty terrible, but the results are pretty funny. Later, when I knew a bit more about what I was doing, I wrote a word generator which draws on a list of words you give it to generate similar words. I've used it to create names and locations for D&D and such.
